Access Methods

ADB (primary — always works)

# Connect to device
adb devices

# Run command inside chroot
adb shell "su 0 chroot /data/local/linux/rootfs /bin/bash -c 'export PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin; YOUR_COMMAND'"

# Push files to device
adb push local_file /sdcard/filename

# Copy into chroot
adb shell "su 0 cp /sdcard/filename /data/local/linux/rootfs/destination/"

SSH (may not work through Tailscale userspace networking)

ssh root@100.70.224.76

Adding a New Service

# Create supervisor config
printf "[program:myservice]
command=/path/to/command
directory=/path/to/workdir
autostart=true
autorestart=true
stdout_logfile=/var/log/server/myservice-out.log
stderr_logfile=/var/log/server/myservice-err.log
" > /etc/supervisor/conf.d/myservice.conf

# Activate
supervisorctl reread && supervisorctl update

Tailscale Funnel / Serve

Current Config

# Check status
tailscale serve status

# Expose a local port publicly via Funnel
tailscale funnel --bg --https 443 http://localhost:5244

# Expose on a different external port
tailscale funnel --bg --https 10000 http://localhost:5000

# Remove a funnel
tailscale funnel --https=443 off

# Reset all
tailscale funnel reset
tailscale serve reset

Boot Process

On device boot:

  1. /system/etc/init/android-server.rc triggers boot wrapper
  2. boot-auto.sh mounts chroot filesystems, enables swap
  3. Inside chroot: starts SSH, cron, Tailscale, Supervisor
  4. Supervisor auto-starts all configured services (including Jaika)

Troubleshooting

pip broken? Always unset Android env vars: unset ANDROID_DATA ANDROID_ROOT ANDROID_BOOTLOGO ANDROID_ASSETS
SSH not connecting? Tailscale uses userspace networking — SSH over Tailscale IP may not work. Use ADB instead.
Service not starting? Check logs: tail -50 /var/log/server/jaika-err.log
PATH issues? Always export: export PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
